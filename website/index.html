Games over QUIC: one-to-one Steam streaming over iroh
Work in progress. Not a finished product.

## why goq

  1. Your key is the address

    The FIDO2 key's hmac-secret seeds the iroh identity. Host and client derive the same node ID from the same physical key. No pairing codes, no accounts.

  2. Headless Sigil host

    A pure-Rust daemon on bare-metal Linux, no desktop, no display. Gamescope gives Steam its own private session; PipeWire feeds a hardware H.264 encoder.

  3. Controller-first

    A virtual Xbox-style gamepad, keyboard, and relative mouse injected via uinput. D-pad PIN pad and a Back+Start escape chord, built for a couch.

  4. Audio included

    PipeWire capture to Opus in bounded datagrams, played back through an AudioWorklet next to the WebCodecs picture.

  5. Bounded everything

    Frame handoff capped at four, decode queue at two, every buffer bounded. Stale frames are dropped, not queued, so latency never drifts.

  6. One host, one client

    A single active client, enforced, over an end-to-end encrypted iroh link, relay-assisted when a direct path isn't possible. No third-party streaming service.

## three taps to steam

  1. 01

    provision sigil

    A bare-metal Linux box. The runbook walks an AMD Bazzite machine from blank to streaming. The daemon runs headless; Gamescope is the display.

  2. 02

    open portal

    In Portal, tap your FIDO2 key and identity is minted on the token. A D-pad PIN pad keeps it couch-friendly.

  3. 03

    tap to play back+start

    Tap the same key anywhere else: Steam appears, your controller just works. Hold Back+Start for a second to escape the session.

You'll need a FIDO2 key with hmac-secret support (YubiKey, Google Titan, …) and a Linux machine for Sigil. Portal builds run on Linux and macOS. Runbook and source →
